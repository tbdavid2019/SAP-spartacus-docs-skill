  如果你要裝 Spartacus 連 SAP Commerce / Hybris 2211，我會先這樣帶你做。

  先決定你是哪一種 2211

  1. 先確認你的後端是 2211.xx 還是 2211-jdk21.x。
  2. 這一步很重要，因為兩者的登入串接方式不同。
  3. 如果你是一般 2211.xx，通常仍走 Resource Owner Password Flow。
  4. 如果你是 2211-jdk21.1 或更新，Spartacus 新版預設走 Authorization Code Flow + PKCE，而且 client 設定也不同。

  第 1 步：先把版本配平

  1. 先不要急著建前端，先確認 Spartacus 版本、Angular 版本、SAP Commerce 版本 是相容的。
  2. 這份 skill 的相容矩陣明確寫到：Spartacus 需要 SAP Commerce Cloud 2105+；而 2211 線的文件也顯示現在有 2211.36、221121.7 這類版本
     線。
  3. 實務上你先選一條線，不要混搭。最穩的是：
      - 舊專案延續：沿用既有 2211.x 線
      - 新專案：直接選同一條 2211 線的最新 patch

  第 2 步：先把 Hybris / SAP Commerce 2211 後端跑起來

  1. 先完成 SAP Commerce 2211 本體安裝。
  2. 確認至少能開：
      - https://localhost:9002
      - Backoffice
      - OCC Swagger，例如 /occ/v2/swagger-ui.html
  3. 這份 skill 裡對 2205 的 backend 安裝流程有明確步驟；2211 沒單獨頁，但初始化思路相同：
      - 準備 commerce 套件
      - 建 recipe
      - initialize
      - start
      - 驗證 Admin Console / Backoffice / OCC

  第 3 步：確認 OCC 真的可用

  1. 在瀏覽器先測這種 API：
      - https://localhost:9002/occ/v2/{baseSite}/basestores/{baseSite}
  2. 如果這個都打不通，先不要進前端。
  3. 前端多數問題，其實是後端 OCC、baseSite、OAuth client、CORS 沒先弄好。

  第 4 步：先處理 OAuth / client 設定

  1. 這一步在 2211 特別重要。
  2. 如果你是 2211.xx（例如 2211.44 這種 JDK17 線）：
      - 依 skill 文件，Spartacus 仍通常使用 Resource Owner Password Flow
      - 你需要在 storefront 把 authorizationCodeFlowByDefault 設為 false
  3. 如果你是 2211-jdk21.1+：
      - 預設應走 Authorization Code Flow
      - client 應設為 Public client credentials
      - 不再用舊式那套 client secret 思維硬接
  4. 這是 2211 最大的分水嶺。沒分清楚，登入一定卡。

  第 5 步：建立 Spartacus 前端

  1. 建 Angular app。
  2. 在專案根目錄執行：

  ng add @spartacus/schematics@<你的版本>

  3. 常見會一起帶的參數：

  ng add @spartacus/schematics@<你的版本> \
    --base-url https://localhost:9002 \
    --base-site electronics-spa \
    --ssr

  4. schematics 會幫你：
      - 安裝 Spartacus 套件
      - 建基本模組
      - 加樣式
      - 設定 cx-storefront
      - 視情況加 SSR

  第 6 步：設定 baseUrl / baseSite

  1. baseUrl 要指向你的 OCC backend。
  2. baseSite 要用你後端真的存在的 site。
  3. 你可以直接在配置裡寫，也可以用 index.html 的 meta tag：

  <meta name="occ-backend-base-url" content="https://localhost:9002" />

  4. 如果 provideConfig() 已經寫了 backend.occ.baseUrl，它會優先於 meta tag。

  第 7 步：處理 CORS

  1. 這是最常見阻塞點之一。
  2. skill 裡的 cors.md 已經把 Spartacus 常用 method/header 列出來了。
  3. 最少先確認：
      - methods 有 GET HEAD OPTIONS PATCH PUT POST DELETE
      - headers 有 origin content-type accept authorization cache-control
      - allowCredentials=true
  4. 如果你有 anonymous consent、personalization、ASM 等功能，還要補對應 header。
  第 8 步：啟動前端驗證

  1. 啟動 Angular / Spartacus app。
  2. 先驗證首頁能不能載入商品與 CMS。
  3. 再驗證登入、加入購物車、checkout。
  4. 順序不要反過來。首頁資料讀不到時就先查：
      - baseUrl
      - OAuth client

  第 9 步：2211 最常見錯誤排查

  1. 401/403：通常是 OAuth flow 選錯，或 client 設定不對。
  2. CORS error：先看 preflight OPTIONS，再補 allowedHeaders / allowedMethods。
  3. 首頁空白但 API 有打：先查 CMS/baseSite/sample data。
  4. 登入頁無法用：先查你到底是 2211.xx 還是 2211-jdk21.x。
  5. ASM 有問題：新版文件提到 221121.1+ 會用到額外的 asm_client。

  我會建議你這樣做

  1. 先給我這 4 個資訊：
      - 你的 SAP Commerce 精確版本，像 2211.44 或 2211-jdk21.1
      - 你的 Spartacus 目標版本
      - 你的 base site 名稱
      - 你是否要用 SAP Commerce 內建 OAuth，還是外部 IdP
  2. 我就可以把上面流程收斂成一版真正可執行的安裝清單，不會只停在概念層。
